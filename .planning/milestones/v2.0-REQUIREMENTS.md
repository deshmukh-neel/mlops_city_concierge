# Requirements: City Concierge v2.0 Production Readiness (ARCHIVED)

**Defined:** 2026-05-21 (post PR #94 merge at `ad8ca84`, after re-baseline)
**Shipped:** 2026-06-03 (PR #100 merged at `14e01dd`)
**Core Value (validated):** A user asks for a constraint-heavy multi-stop itinerary in natural language and the agent returns a coherent plan grounded in real places — geographically anchored, temporally consistent, constraint-satisfying — with a booking deep-link.

**Scope (delivered):** Built the eval infrastructure that lets every subsequent fix be
scored against committed baselines across multiple providers, then fixed the three
agent-behavior bugs surviving the PR #94 reliability merge (category compliance,
rationale-stop alignment, minimal-edit refinement). All 21 requirements landed; one
gate (D-06-09 part 2) accepted-with-notes because the pre-Phase-6 baseline was a
fail-open false positive — the real fix is scoped to v2.1.

## Deferred to v2.1 (or later)

Carried forward — these were never in v2.0 scope, or were observed during v2.0 but couldn't be remediated within v2.0 architecture:

| Requirement | Reason |
|---|---|
| **STEP-01**: dynamic `max_steps` for revision turns | Original v2.0 phase 6 — dropped after re-baseline didn't reproduce the step-limit short-circuit (the earlier observation was the JSON-safety crash misattributed). |
| **NOJ-01**: N-of-M pass rate (`--runs-per-case N`) | Statistical rigor under temperature=1.0 model variance. Defer until baselines are stable. |
| **JUD-01**: LLM-as-judge scorer for rationale quality | Add only if deterministic checks miss too many real failures. Requires a different model family for judge vs. agent (P3). |
| **OVR-07**: Per-PR ephemeral MLflow aliases | `RAG_MODEL_OVERRIDE` proved sufficient for v2.0. |
| **OVR-08**: `/reload-model` admin endpoint | Restart is acceptable for now. |
| **CLO-01**: Investigate over-aggressive closure detection on Mission queries | Flagged during re-baseline. Pre-Phase-3 D-07 check confirmed `place_is_open` is correct; remaining anomaly may be hours-data drift or `_per_stop_closure_status` overly cautious. |
| **REASONING-01..04** (v2.1 scope): reasoning-content thread-through in `_prune_for_llm`; prompt/rubric decoupling; Anthropic provider wiring; honest cross-model baseline regen | New requirement class surfaced during Phase 6 gpt-5-mini / gpt-5.4-mini probe. All reasoning models currently fail on this codebase's tool-loop tasks because reasoning state is dropped across turns. Without this, every new model the field ships in 2026 is unusable here. |

## Out of Scope (v2.0)

| Feature | Reason |
|---------|--------|
| DSPy / managed eval UI | User explicitly ruled out; eval harness is intentionally a Python script |
| Streaming responses, retry policies, rate limiting | Separate hosting concerns; v2.0 is correctness, not throughput |
| Full LLM-as-judge by default | Bias risk (P3); add only if deterministic checks plateau |
| Bootstrap CIs / Wilson intervals | Overkill until stable baselines exist |
| MLflow-backed per-run eval storage | Local JSON baselines are sufficient for v2.0 cross-PR diffing |
| Real-time eval dashboards (W&B, LangSmith) | No separate service hosting in scope |
| Booking automation (Playwright against Resy/Tock) | Pre-existing project-level deferral |
| Dropping v1 embeddings | Pre-existing W6-gated deferral |
| LLM-extracted KG edges | Pre-existing W7 deferral |
