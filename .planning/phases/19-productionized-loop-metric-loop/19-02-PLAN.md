<objective>
Extend the existing idempotent sandbox provisioner to build a POPULATED baseline (D-01,
D-02). `--populated` is BOTH the build AND the idempotent reset: it DROP+recreates the DB,
rebuilds the schema, INGESTS the broad real-SF non-gap catalog, and fully embeds it BEFORE
any before-snapshot — restoring the EXACT baseline data and embeddings every run. The gap
bucket is EXCLUDED from ingest by marking ONLY the gap-bucket queries 'completed' (ingest
SKIPS 'completed' queries), so the gap stays un-ingested and genuinely under-served at the
before-snapshot. A bare `--reset` is a SCHEMA-ONLY alias (DROP+recreate+schema, no
ingest/embed). Add a `make sandbox-provision-populated` target mirroring the existing
`make sandbox-provision` env-guard. Reuse every existing prod-safety guard unchanged.

CRITICAL INVERSION (do NOT re-copy Phase 16's direction): Phase 16 (empty sandbox) marked
the non-gap catalog 'completed' so ingest ingested ONLY the gap. Phase 19 wants the OPPOSITE
— the POPULATED "before" corpus IS the broad non-gap catalog (ingested + embedded), and the
gap bucket is the EXCLUDED/under-served set. Therefore mark ONLY the gap-bucket exclusion set
'completed' (so ingest skips the gap), then ingest the rest, then embed.

Purpose: Phase 16's loop measured hit@k against an EMPTY sandbox (before=0 by
construction, no retrieval competition). Phase 19's honest capstone measures a before→after
lift on a NON-empty baseline where the new gap places compete against the existing corpus.
That requires a reproducible populated "before" state where the chosen gap bucket is
genuinely under-served, and a reset that fully restores it. `--populated` IS that reset:
every invocation DROP+recreates, re-ingests the non-gap catalog, and re-embeds, so the
baseline is byte-identical run to run (embed-v2 skips already-current rows, so a partial
reset that only cleared proposals would leave an embed backlog that pollutes the after-diff
— D-02 LOCKED CONSTRAINT; that is why the populated reset re-runs ingest+embed, not just a
schema rebuild).

Output: `provision_sandbox.sh` gains `--populated` + `--reset`; `Makefile` gains
`sandbox-provision-populated`; a unit test asserts the shell flag-routing + guard ordering
without running live ingest.
</objective>
