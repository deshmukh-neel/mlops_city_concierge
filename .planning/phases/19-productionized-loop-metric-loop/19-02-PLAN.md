<objective>
Extend the existing idempotent sandbox provisioner to build a POPULATED baseline (D-01,
D-02): a `--populated` mode that ingests a real SF subset EXCLUDING the gap bucket and
fully embeds it BEFORE any before-snapshot, and a `--reset` mode that DROP+recreates the
DB and restores the EXACT baseline data and embeddings (not just clears proposals). Add a
`make sandbox-provision-populated` target mirroring the existing `make sandbox-provision`
env-guard. Reuse every existing prod-safety guard unchanged.

Purpose: Phase 16's loop measured hit@k against an EMPTY sandbox (before=0 by
construction, no retrieval competition). Phase 19's honest capstone measures a before→after
lift on a NON-empty baseline where the new gap places compete against the existing corpus.
That requires a reproducible populated "before" state where the chosen gap bucket is
genuinely under-served, and a reset that fully restores it (embed-v2 skips already-current
rows, so clearing only proposals leaves an embed backlog that pollutes the after-diff —
D-02 LOCKED CONSTRAINT).

Output: `provision_sandbox.sh` gains `--populated` + `--reset`; `Makefile` gains
`sandbox-provision-populated`; a unit test asserts the shell flag-routing + guard ordering
without running live ingest.
</objective>
